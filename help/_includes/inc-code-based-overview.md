# コードベースのエクスペリエンス – 概要

## もう一度

コードベースのエクスペリエンスにより、JavaScript、API、テンプレート言語などのカスタムコードを使用して、Adobe Journey Optimizer内でパーソナライズされた動的なコンテンツとメッセージを配信できます。

このチャネルは、標準的なビジュアルオーサリングツール（WYSIWYGエディター）に留まらず、詳細にカスタマイズされたデータ主導のインタラクションを構築するのに最適です。これにより、開発者と最適化マネージャーは、独自のパーソナライゼーションロジックとコンテンツレイアウトを柔軟に設計できます。

例：

* 複雑なユーザー属性や行動にもとづいて、メールやプッシュ通知のコンテンツをカスタマイズします。
* サードパーティ APIを統合してリアルタイムのデータを取得し、メッセージに注入する。
* 特定のイベントによってトリガーされる、高度な意思決定ロジックやマルチステップのキャンペーンを構築する。

**重要：**&#x200B;正しいデータ処理と配信を確実に行うには、開発者のスキルとテストが必要です。

## ユースケース

必要なときにコードベースのエクスペリエンスを選択：

| # | 利点 | 理由 | ユースケースの例 |
|---|---------|-----|-------------------|
| 1 | 高度なパーソナライゼーション | 複雑なロジックとリアルタイムデータを使用して、高度にカスタマイズされたメッセージを作成できる | <ul><li>最近の閲覧履歴にもとづく動的な商品レコメンデーション</li><li>ユーザーロイヤルティステータスから計算されたパーソナライズされた割引</li></ul> |
| 2 | 外部システムの統合 | コミュニケーションで外部データを直接取得して表示 | <ul><li>天候ベースのプロモーション</li><li>電子メールでリアルタイムに在庫を更新</li></ul> |
| 3 | 高度なワークフローの作成 | 利用者のアクションごとに適応するマルチステップの条件付きジャーニーを構築します | <ul><li>タイムリーなリマインダーによるカート放棄のフォローアップ</li><li>非アクティブのしきい値の後にトリガーされる再アクティブ化キャンペーン</li></ul> |
| 4 | プラットフォームの限界を超える | すぐに使用できる機能またはメッセージ形式はサポートされていません | <ul><li>カスタムインタラクティブメール要素</li><li>複雑なSMS メッセージのバリエーション</li></ul> |
| 5 | キャンペーンの柔軟性を向上 | プラットフォームの更新を待つことなく、ロジックやコンテンツをすばやく調整 | <ul><li>季節別コンテンツ切り替え</li><li>ジャーニーにおけるリアルタイムのエラー処理</li></ul> |

## コードベースのエクスペリエンスを利用しない場合

* ノーコードツールで構築できる、迅速でシンプルなキャンペーンが必要な場合。
* 開発者リソースやテスト環境にアクセスできない場合。
* タイムラインや予算の制約によってカスタム開発が制限される場合。
* プラットフォームの機能に合わせた標準的なメッセージを送信できます。
* メンテナンス性が懸念される場合、カスタムコードには継続的なサポートが必要です。
