# アプリ内メッセージ

## アプリ内メッセージとは？

オーディエンスが積極的にエンゲージメントしている間、アプリ内メッセージはモバイルアプリ内に表示されます。 これらのオーバーレイスタイルのメッセージは、バナー、ポップアップ、またはカードとしてアプリインターフェイスの上に直接表示されます。 ロック画面やアプリ環境外には表示されません。

アプリ内メッセージは、特定の画面の表示、購入の完了、ターゲットを絞ったオーディエンスセグメントの選定など、特定のユーザーのアクションや条件によってトリガーされます。


以下に例を示します。

* ユーザーが新しい機能をロック解除すると、ポップアップが表示されるゲーム。
* 利用者が商品を閲覧している間に、クーポンコード付きのバナーを表示するショッピングアプリ。
* 現在のアクティビティにもとづいて、フォローする新しいアカウントを提案するソーシャルアプリ

## ユースケース

| **メリット** | **なぜ** | **使用例** |
|----------------------------------|------------------------------------------------------------------------|----------------------------------------------------------------------------------------|
| ユーザーエンゲージメントの向上 | ユーザーがアプリでアクティブな間にキャプチャし、より高い可視性とアクションを実現します | <ul><li>オンボーディングのウォークスルー</li><li>機能のお知らせ</li><li>リアルタイムのサポートポップアップ</li></ul> |
| コンテクストに関連 | ユーザーの行動に基づいてメッセージをトリガーし、タイムリーでパーソナライズされた | <ul><li> 機能を利用した後のアップセル</li><li> 非アクティブなナッジ</li><li> エラーまたは成功メッセージ</li></ul> |
| リアルタイム配信 | 時間の制約を受けるアップデートや重要なアップデートを迅速に提供 | <ul><li> システム停止</li><li>トランザクション確認</li><li>支払い失敗</li></ul> |
| 外部チャネルに依存しない | 電子メールやSMSを使用することなく、製品内でユーザーエクスペリエンスを維持できます | <ul><li> チュートリアル完了リマインダー</li><li>体験版の有効期限に関するアラート</li></ul> |
| コンバージョンの可能性の向上 | インコンテクストに配置されたメッセージは、オフプラットフォームのメッセージよりもコンバージョン率が高まります | <ul><li> プランの制限に達した後のアップグレードプロンプト</li><li>アプリ内調査</li></ul> |
| カスタマイズとセグメンテーション | ユーザーデータやアプリイベントにもとづいてパーソナライズ可能 | <ul><li> ユーザー層やアクティビティレベルに合わせたメッセージ</li><li> 役割に特化したアラート </li></ul> |
| 非侵入的（適切に設計された場合） | ユーザーフローを損なうことなく、微妙でありながら効果的なコミュニケーションが可能 | <ul><li> ツールヒント</li><li>更新付きスライドイン</li><li>チャットウィジェットのナッジ</li></ul> |


## アプリ内通信を使用する&#x200B;*not*&#x200B;の場合

* **ユーザーはアプリをアクティブに使用していません**\
  ユーザーが現在ログインしていない場合や、製品とエンゲージしていない場合、アプリ内メッセージは表示されません。 代わりに電子メールまたはプッシュ通知を使用します。
* **重大な問題または時間の制約を受ける問題で、すぐに対処する必要がある**\
  緊急メッセージ（セキュリティアラート、支払い失敗など）には、SMSやメールなど、アプリ外でユーザーにアプローチできるチャネルを利用します。
* **規制上または法律上のコミュニケーション**\
  コンプライアンス関連のメッセージ（条件の更新やポリシーの変更など）には、配信や確認の読み取りが必要になる場合があります。メールに適しています。
* **アカウントの再アクティブ化またはウィンバック キャンペーン**\
  非アクティブなユーザーや解約したユーザーには、アプリ内メッセージは表示されません。 電子メールやプッシュ通知を通じたリエンゲージメント施策の実施：
* **大量のトランザクション更新**\
  アップデートや領収書の発送などの通知は、アーカイブと利便性のために、電子メールを通じて送信する方が適しています。
* **過剰な使用はバナーのブラインドネスや苛立ちにつながります**\
  アプリ内メッセージが多すぎると、ユーザーエクスペリエンスに悪影響を及ぼし、フラストレーションやメッセージの無視につながる可能性があります。
* **接続なし/オフラインのシナリオ**\
  アプリ内メッセージは、オンラインとセッション中の情報を提供します。オフラインファースト環境では役に立ちません。
